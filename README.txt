마법소녀 슈팅: 던전 탐험 FPS
1. 프로젝트 개요
이 프로젝트는 90년대 고전 FPS의 감성을 현대적인 로그라이크 요소와 결합한 2.5D 레트로 FPS 게임입니다. 순수 JavaScript(Vanilla JS)와 HTML Canvas를 사용하여 개발되었으며, 별도의 게임 엔진 없이 레이캐스팅(Raycasting) 기법을 직접 구현하여 3D 환경을 렌더링합니다.

핵심 목표는 단순한 슈팅을 넘어, 무작위로 생성되는 던전을 '탐험'하고, 성장하며, 더 깊은 곳으로 도전하는 재미를 제공하는 것입니다.

2. 핵심 아키텍처: 상태 중심 설계
이 프로젝트는 중앙 집중식 상태 관리라는 매우 현대적이고 효율적인 아키텍처를 채택하고 있습니다.

Script/state.js: 모든 것의 중심입니다. 이 파일은 게임의 모든 동적인 데이터(플레이어 정보, 적 목록, 맵 구조, DOM 요소, 로드된 에셋 등)를 담고 있는 '단일 진실 공급원(Single Source of Truth)' 역할을 합니다.

나머지 모듈 (gameLogic.js, render.js 등): 이 모듈들은 자체적으로 상태를 거의 가지지 않습니다. 대신 state.js의 데이터를 읽어 로직을 처리(읽기)하고, 그 결과를 다시 state.js에 기록(쓰기)하는 방식으로 동작합니다.

이러한 설계는 다음과 같은 큰 장점을 가집니다.

예측 가능성: 게임의 모든 상태가 한곳에 모여있어 데이터 흐름을 추적하기 쉽습니다.

디버깅 용이성: 버그가 발생했을 때, state.js의 값이 어떻게 변하는지만 관찰하면 원인을 쉽게 찾을 수 있습니다.

모듈 분리: 각 모듈은 자신의 역할(로직, 렌더링, 입력 등)에만 집중할 수 있어 코드가 깔끔하게 유지되고 기능 확장이 용이합니다.

3. 주요 스크립트 파일 분석 (Script/)
파일명

역할

핵심 기능 및 설명

main.js

게임 오케스트레이터 (진입점)

게임의 시작과 끝을 관리합니다. init()을 통해 모든 모듈을 초기화하고, requestAnimationFrame으로 메인 게임 루프(gameLoop)를 실행하며, startGame, gameOver, nextFloor 등 게임의 전체 흐름을 제어합니다.

state.js

중앙 상태 저장소

게임의 살아있는 데이터베이스. 플레이어, 적, 아이템, 맵, 파티클 등 모든 동적 데이터를 전역적으로 저장하고 관리합니다.

constants.js

설정 및 밸런스 데이터

게임의 모든 정적 상수(에셋 경로, 적 스탯, 무기 성능, 맵 크기 등)를 정의합니다. 이 파일의 값만 수정해도 게임의 밸런스와 콘텐츠를 쉽게 조정할 수 있습니다.

render.js

렌더링 엔진 (그래픽)

화면에 보이는 모든 것을 그립니다. loadAssets로 에셋을 로드하고, render 함수에서 레이캐스팅으로 벽/바닥/천장을, Z-버퍼를 이용해 스프라이트를 렌더링합니다. 저해상도 오프스크린 캔버스를 사용해 성능을 최적화합니다.

gameLogic.js

게임의 두뇌 (로직)

update 함수를 통해 매 프레임 게임 월드를 살아있게 만듭니다. 플레이어 이동과 충돌 처리, 적 AI(BFS 경로 탐색), 발사체 물리, 공격 판정 등 모든 게임 규칙을 처리하고 state.js를 업데이트합니다.

input.js

사용자 입력 처리

키보드, 마우스(포인터 잠금), 모바일 터치(가상 조이스틱) 입력을 모두 감지하고, 이를 gameLogic.js가 이해할 수 있는 통일된 데이터 형식(예: forward: 1)으로 변환합니다.

ui.js

HUD 및 UI 관리

state.js의 데이터를 기반으로 HTML DOM 요소를 업데이트합니다. 플레이어 HP, 탄약, 상태 얼굴 이미지 변경, 무기 교체 애니메이션 등 모든 UI 관련 시각 효과를 담당합니다.

mapGenerator.js

레벨 디자이너

'방과 복도(Room-and-corridor)' 알고리즘을 사용하여 무작위 던전 맵(S.map 2차원 배열)을 절차적으로 생성합니다. 플레이어 시작 지점과 출구 위치를 결정합니다.

4. 데이터 및 에셋 관리 (Image/, Data/)
스프라이트 시스템: 모든 이미지는 스프라이트 시트(*.png)와 각 스프라이트의 좌표 정보를 담은 아틀라스 파일(*_data.json)의 쌍으로 이루어져 있습니다. 이 방식은 이미지 로딩 효율을 높여줍니다.

테마 시스템: Data/dungeon_progression.json 파일은 각 층의 테마(예: main, cave)와 변형(variation)을 정의합니다. render.js는 이 정보를 바탕으로 해당 층에 맞는 벽, 바닥, 천장 텍스처를 동적으로 불러와 사용하여 다채로운 던전 분위기를 연출합니다.

5. 게임 실행 흐름
초기화: main.js가 DOM 요소를 준비하고 모든 이벤트 리스너를 설정합니다.

에셋 로딩: 사용자가 'START' 버튼을 누르면 render.js의 loadAssets가 constants.js에 정의된 모든 이미지와 사운드를 비동기적으로 불러옵니다.

게임 시작:

main.js의 startGame 함수가 호출됩니다.

mapGenerator.js가 첫 번째 층의 맵 데이터를 생성합니다.

gameLogic.js가 생성된 맵에 적들을 스폰합니다.

게임 루프 (gameLoop) 시작:

입력: input.js가 사용자의 조작을 감지합니다.

업데이트: gameLogic.js가 입력과 deltaTime을 바탕으로 플레이어, 적, 발사체 등의 상태를 업데이트하고 그 결과를 state.js에 반영합니다.

렌더링: render.js가 업데이트된 state.js의 정보를 읽어 3D 월드와 스프라이트를 캔버스에 그립니다.

UI 업데이트: ui.js가 state.js의 정보를 읽어 화면의 HP, 탄약 등의 HUD를 업데이트합니다.

위 과정이 requestAnimationFrame을 통해 매 프레임 반복됩니다.
